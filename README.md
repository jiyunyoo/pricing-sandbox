# 🧪 Pricing Sandbox — 제로 리스크 프라이싱 시뮬레이터

> **사람들 대신, AI에게 가격을 올려 묻다.**
> 통계청 데이터로 만든 가상 소비자(호모 실리쿠스)에게 가격 인상 시나리오를 제시하고, **재화 특성**과 **커뮤니티 토론**이 가격 저항을 어떻게 바꾸는지 측정하는 인터랙티브 데모.

**▶ 라이브 데모: [jiyunyoo.github.io/pricing-sandbox](https://jiyunyoo.github.io/pricing-sandbox/)**

KAIST 정보경영석사(IMMS) 졸업 프로젝트 · 김기환 · 이민성 · 유지윤 · 2026

---

## 📌 한 줄 요약

기업이 가격을 올리는 결정은 매출을 좌우하는 고위험 선택이지만, 실제로 올려 보기 전에는 결과를 알기 어렵습니다. 이 프로젝트는 한국 통계청 데이터로 정교하게 구성한 **AI 가상 소비자**에게 가격 인상을 물어, 실제 매출 손실 없이 가격 정책을 사전 점검하는 **"제로 리스크 프라이싱 샌드박스"** 를 제안하고 구현합니다.

---

## 🎯 핵심 연구 질문

| | 질문 |
|---|---|
| **RQ1** | 재화의 정보 구조(탐색재 vs 경험재)에 따라 가격 저항이 다르게 나타나는가? |
| **RQ2** | 커뮤니티 토론은 가격 저항을 증폭(정보 캐스케이드)하거나 완화(밴드왜건)하며, 그 방향은 재화 특성에 따라 달라지는가? |

- **탐색재(Search good)** — 스펙으로 즉시 비교 가능, 가격 민감 *(예: 27인치 모니터)*
- **경험재(Experience good)** — 써 봐야 가치를 앎, 브랜드·후기 의존 *(예: 스킨케어 앰플)*

## 🔑 핵심 결과 (+20% 인상 · 토론 활성화)

같은 +20% 인상에도 두 재화는 **정반대로** 반응했습니다.

| 지표 | 🖥️ 모니터 (탐색재) | 🧴 화장품 (경험재) |
|---|---|---|
| **가격 탄력성 ε_sim** | **−2.25** (탄력적) | **−0.48** (비탄력적) |
| **구매율 (기준가 → 토론 후)** | 75% → 35% | 68% → 64% |
| **구전 승수 WOM_m** | **1.19** (캐스케이드·증폭) | **0.56** (밴드왜건·완화) |
| **의사결정 번복 (FLIP)** | 1명 (수빈: 구매→포기) | 1명 (영자: 포기→구매) |
| **오피니언 리더** | 종호 (절약 가장) | 도윤 (럭셔리 전문직) |

> **해석.** 탐색재는 "더 싼 대안이 있다"는 정보가 토론으로 빠르게 퍼져 구매 포기가 연쇄적으로 일어납니다(WOM_m > 1, 정보 캐스케이드). 반대로 경험재는 브랜드 충성과 "남들도 산다"는 동조 심리로 가격 저항이 누그러집니다(WOM_m < 1, 밴드왜건). **가격 인상 전략은 재화 특성에 따라 달라야 하며, 특히 탐색재는 커뮤니티 여론 관리가 가격 정책의 성패를 좌우합니다.**

## 🖥️ 데모 사용법

1. **실험 설계** — 재화 종류(탐색재/경험재), 가격 인상폭(−20% ~ +50%), 커뮤니티 토론 여부를 설정
2. **시뮬레이션 실행** — 16명의 가상 소비자가 ① 개별 1차 판단 → ② 커뮤니티 토론 → ③ 토론 후 2차 판단의 단계를 거침
3. **결과 확인** — 가격 탄력성(ε_sim), 구전 승수(WOM_m), 구매율 변화, 의사결정 번복(FLIP), 수요 곡선이 대시보드에 생성됨

페르소나 카드를 클릭하면 LLM에 입력되는 **시스템 프롬프트 페이로드**와 **출처 통계**를 확인할 수 있습니다.

## 📐 측정 지표

**가격 탄력성 (Price Elasticity)**
```
ε_sim = (%ΔQ) / (%ΔP) = [(Q(1.2·P₀) − Q(P₀)) / Q(P₀)] / 0.20
```
`|ε| > 1`이면 탄력적(elastic), `|ε| < 1`이면 비탄력적(inelastic).

**구전 승수 (Word-of-Mouth Multiplier)**
```
WOM_m = ΔQ_post / ΔQ_pre
```
`> 1`이면 토론이 가격 저항을 증폭(정보 캐스케이드), `< 1`이면 완화(밴드왜건).

**의사결정 번복 (McNemar 검정)**
```
χ² = (|b − c| − 1)² / (b + c)
```
토론 전후 페어드 데이터의 변화 유의성(χ² > 3.84, α=0.05) 검정.

## ⚙️ 기술 사양

- **단일 HTML 파일** — 모든 로직·디자인·삽화(SVG)가 하나의 파일에 포함되어 설치·서버 없이 오프라인에서도 동작
- **API 키 불필요** — 라이브 시연의 안정성과 재현성을 위해 **결정론적(deterministic) 엔진**으로 동작하여 항상 동일한 결과를 재현
- **벡터 기반 시각화** — 16개 페르소나 캐릭터 일러스트와 차트(막대그래프·수요 곡선)를 모두 인라인 SVG로 렌더링하여 어떤 화면에서도 선명
- **의존성 없음** — 외부 프레임워크 없이 순수 HTML/CSS/JavaScript

### 데모 엔진 vs 본 실험

| | 데모 (이 저장소) | 본 실험 (Full Study) |
|---|---|---|

| 규모 | 16명 × 1회 시연 | 16명 × 2×2 × **N=30** = 1,920 trials |

## 🚀 로컬에서 실행하기

```bash
git clone https://github.com/jiyunyoo/pricing-sandbox.git
cd pricing-sandbox
open index.html        # macOS  (Windows: start index.html · Linux: xdg-open index.html)
```

`Settings → Pages → Build and deployment → Deploy from a branch → main / (root) → Save`. 이후 `index.html`을 갱신하여 push하면 1~2분 내 자동 재배포됩니다 (저장소는 공개 상태여야 함).
